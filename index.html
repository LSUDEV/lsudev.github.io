RIZ-100 - A 64-Bit Expandable Micro Computer

Designed to be the 2nd generation of the Minature SRC Line-up
Still a work in progress, so changes may not be visible on Github.
Github Here

  • Expanded ALU archtecture, now supporting hardware level Floating Point archtecture
  • Hybrid & Dedicated Storage Controller, for communicating with Cache, Ram & Hard Storage
  • Fluid Exchange ports with it's own dedicated registers for handling both asyncronous and interruptable processing
  • Internal and Easy to expand extensions, allowing hardware level expansions like SHA-1 & IO Serializers
  • 64-bit instruction processor handling up to 32 General Registers, with an additional 8 Special Registers
  • No more cursed jump storage to R15, there is now a dedicated "J" register for jumps
  • And of course a joke operator, cause why not, "riz RA, RB"
Here's the new instruction format:

TYPE Default | 63:56 OP | 56:51 A | 50:45 B | 45:39 C
- Regular format of A, B, C selection operation
- Ex: add R1, R2, R3 -> R1 = R2 + R3

TYPE Immediate | 63:56 OP | 56:51 A | 50:45 B | 44 SIGN | 43:0 IMM
- Immediate format of A, B, IMM selection operation
- Ex: addi R1, R2, 0x1000 -> R1 = R2 + 4096

TYPE Conditional | 63:56 OP | 56:51 A | 50:45 CONDITION | 44 SIGN | 43:0 IMM
- Immediate format of A, B, IMM conditional operation
- Ex: br[nz] R6, 25 -> if R6 != 0, then branch by 25 steps

MSRC-1 - A small but capable 32-bit microchip

Its a computer, literally.
Have fun reading all the comments, shows how much I totally like this thing.
No I am not converting this to silicon.
Yes, this is designed to run at 50 Mhz cry about it.
Github Here

  • Support for basic ALU instructions, including HI/LO registers for 64-bit operations.
  • 2048-byte RAM module, 32-bit rows by 512, supporting read/write operations, acting as the primary executable.
  • Support for I/O to external devices supporting 32-bit low or 64-bit low & high.
  • A simple state machines, no complications, a simple control unit for all operations.
  • Built-in debuggers for learning how the system operations via ModelSim.
  • Operators in seperate files, for easy traversal and additions of new operators to the system, supporting up to 32 operators.

MWB - Modulus Weapon Base

An attempt to create the most realistic weapon system while giving content creators an easier time working with it. As you may guess by the name, the attachment system on this is made to be ludicrously customizable, to the point where you can even change the bullet's lua physics engine.

Some key features:

  • Recoil is lua based, so you can customize it however you want.
  • Attachments are multivariable and managed automatically, meaning you can change even things like SWEP.Primary.Stick.Config.Nuts to a different value and you won't have to worry about restoring it.
    Here's a look at the layout here: https://pastebin.com/bJcZ8v1b
  • Weapons are affected by the direction of your movement, so spread is kind of out of the question...
  • Scopes go off the 2 point method, meaning I tried to make it as realistic as possible.
  • Physical bullets, using kinematic equations and highly customizable for the attachment system. (Like changing the velocity, color or even the physics behind it)
  • Embedded hooks, yes hooks in a weapon base... For customizations, like
    SWEP:AddHook("BulletFired", "ChangeColor", function(self, bullet) bullet:SetColor(gray) end)
  • The directions the bullet take is dependant on the angle of the weapon, this is not done client-side as it would give the client too much control...
    Instead it's using the same concept of source engine, and running it shared in prediction.
